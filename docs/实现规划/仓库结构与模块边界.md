# 仓库结构与模块边界

## 设计原则

- 主仓库独立维护，避免直接魔改外部基线仓库。
- ESAM、OnlineAnySeg、GASP 只作为 `third_party/` 参考与依赖。
- 代码结构按数据流和两层图拆分，而不是按论文章节随意堆叠。

## 顶层目录

- `configs/`: 配置文件
- `data/`: 数据集入口
- `cache/`: 中间缓存
- `third_party/`: 外部仓库
- `src/ctga/`: 主代码
- `tools/`: 预处理和调试工具
- `tests/`: 测试
- `docs/`: 文档主入口

## `src/ctga/` 模块划分

- `common/`: 类型、几何、IO、配置、日志
- `datasets/`: posed RGB-D sequence 数据层
- `frontends/`: 2D masks 与 embedding 读取
- `mapping/`: active map、frustum query、track reprojection
- `primitives/`: 几何 over-segmentation 与 primitive feature
- `graph1_evidence/`: Layer-1 evidence graph 建图、打分、signed graph 组装、求解
- `graph2_match/`: Layer-2 unary / relation feature、candidate gating、图匹配求解
- `memory/`: track bank、feature fusion、生命周期
- `supervision/`: GT 投影、边标签、teacher replay
- `losses/`: stage-1 / stage-2 损失
- `trainers/`: 各阶段训练入口
- `inference/`: online engine、评估与诊断
