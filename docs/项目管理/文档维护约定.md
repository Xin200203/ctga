# 文档维护约定

## 原则

- 方法变化优先更新 `docs/方法/`
- 工程拆分变化优先更新 `docs/实现规划/`
- 关键假设和风险变化优先更新 `docs/理论分析/`
- 实验指标、baseline、ablation 变化优先更新 `docs/实验设计/`
- 开发状态和任务拆分优先更新 `docs/项目管理/`

## 记录方式

- 每个文档尽量只承担一种职责，避免一份文件同时承载方法、实现、实验、会议纪要。
- 原始长讨论尽量归档到 `docs/归档/`，正式结论再沉淀到主题文档。
- 后续新增文档时，优先先补 `docs/README.md` 的导航链接。
- 根目录原则上不再放研究长文档；原始材料统一进入 `docs/归档/`。
