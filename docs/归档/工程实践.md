我先说明三条已经核对过的工程事实，这会直接影响你的实现路线：

第一，ESAM 已经给了比较干净的在线接口：自定义数据目录是 color/ depth/ pose/ intrinsic.txt，vis_demo/stream_demo.py 里 StreamDemo.run_single_frame(color_map, depth_map, pose, intrinsic=None) 会返回 all_points, points_color, pred_ins_mask。这非常适合拿来做在线输入脚手架，但不适合直接承载你新的图求解主干。 ￼

第二，OnlineAnySeg 已经把“空间候选生成”这件事做得很具体了：仓库主文件就是 main.py / Scene_rep.py / MaskGraph.py / voxel_hashing.py；其 VoxelHashTable.query_mask_w_mapping 用体素哈希查重叠历史 mask，MaskGraph.query_mask_under_visible_part 计算 visibility-aware 的双向 contain ratio，Scene_rep 维护 contain_matrix、global2merged_mask_id、mask feature / geo feature，并按 merge_mat 聚类合并。作者还在 README 里明确把“用自适应方法替换 threshold-based merging”列为待改进项。 ￼

第三，GASP 已经给了现成的 signed graph partition 接口 run_GASP(graph, signed_edge_weights, linkage_criteria=..., add_cannot_link_constraints=...)；它对应的是 signed graph 上的 multicut / correlation clustering 近似求解。另一方面，GMTracker 证明了“对象间关系进入二阶 graph matching + gating/GST 限制搜索空间”是可行的；ODAM 则证明了 posed 视频里的 frame-to-model association 本身就可以是一个图问题。SceneGraphFusion 和 SAI3D 又分别证明了“primitive 节点 + same-part / sparse affinity + merge”这类第一层图在 3D 场景里是成立的。 ￼

A. 实现总纲

A.1 目标边界

这版实现文档对应的 v1 系统目标 是：

输入：posed RGB-D sequence + 预计算的 2D class-agnostic masks / embeddings
输出：online class-agnostic 3D object tracks
核心创新：
	1.	同帧 2D mask – 3D primitive – history track 的证据图
	2.	当前对象图到历史 track 图的二阶匹配
	3.	不把 open-vocabulary semantics 混进主关联环节

这意味着：
	•	先不做语义主导关联
	•	先不做端到端可微 graph partition / QAP
	•	先不做长时 2D track memory
	•	先不做SLAM 后端联合训练

A.2 两层图在工程里的最小闭环

层 1：Evidence Graph

输入：
	•	当前帧 2D masks
	•	当前 active frustum 3D primitives
	•	当前可见的 active tracks

输出：
	•	当前帧局部对象 CurrentObjectHypothesis[]

目标：
	•	修复 over-seg / under-seg

层 2：Association Graph

输入：
	•	当前局部对象
	•	active track bank

输出：
	•	matched / newborn / dormant / reactivate

目标：
	•	稳定 ID
	•	降低 ID switch
	•	在重复物体场景里利用二阶关系 disambiguation

	•	对每条对象边 (r,s) 只计算 K_r × K_s 的兼容矩阵

E.3 产物 2：primitive GT 归属

PrimitiveGTAssigner

class PrimitiveGTAssigner:
    def assign(
        self,
        primitives: list[Primitive3D],
        gt_instance_voxel_ids: dict[int, torch.LongTensor],
    ) -> dict[int, int | None]:
        """
        prim_id -> gt_instance_id or None
        """

推荐标签规则：

对于每个 primitive p_j，计算它与所有 GT instance 的 overlap。
若满足：
	•	overlap(p_j, gt*) / |p_j| >= 0.5
	•	且它覆盖其他 GT 的泄漏比例不超过 0.1

则赋给最佳 GT instance；否则标为 ignore。

这个阈值直接借鉴 SceneGraphFusion 的 same-part 生成规则：它也是先把几何 segment 与 GT object 匹配，要求交并覆盖足够大，同时不能明显覆盖别的 GT object。 ￼

          y_mp: [E_mp] in {-1,0,1}

标签规则：

	•	PT：primitive GT == teacher track GT → 正；不同 → 负
	•	MT：mask GT == teacher track GT → 正；不同 → 负

其中 -1 表示 ignore。

L_total = λ_mp L_mp + λ_pt L_pt + λ_mt L_mt + λ_pp L_pp

G. 在线推理伪代码

这是最重要的运行时主循环。

class OnlineEngine:
    def __init__(self, cfg):
        self.map = ActiveMap(cfg.voxel_size)
        self.mask_cache = OfflineMaskCache(cfg.mask_cache_root)
        self.primitive_builder = PrimitiveBuilder(cfg.primitive)
        self.evidence_graph_builder = EvidenceGraphBuilder(cfg.graph1)
        self.edge_scorer_l1 = EdgeScorerL1(cfg.graph1)
        self.signed_graph_assembler = SignedGraphAssembler(cfg.graph1)
        self.layer1_solver = GaspPartitionSolver(cfg.graph1)
        self.current_object_builder = CurrentObjectBuilder(cfg.graph1)

        self.assoc_graph_builder = AssociationGraphBuilder(cfg.graph2)
        self.unary_scorer = UnaryScorer(cfg.graph2)
        self.relation_scorer = RelationScorer(cfg.graph2)
        self.candidate_gater = CandidateGater(cfg.graph2)
        self.component_builder = ComponentBuilder(cfg.graph2)
        self.graph_matcher = BeamQAPSolver(cfg.graph2)
        self.hungarian_fallback = HungarianFallbackSolver(cfg.graph2)

        self.track_bank = TrackBank(cfg.memory)
        self.track_renderer = VisibleTrackRenderer(cfg.mapping)

    @torch.no_grad()
    def step(self, frame: FramePacket):
        # A. integrate current frame
        self.map.integrate(frame)
        active = self.map.query_active_frustum(frame)

        # B. current observations
        masks = self.mask_cache.load_masks(frame.scene_id, frame.frame_id)
        primitives = self.primitive_builder.build(
            active["active_points_xyz"],
            active["active_points_rgb"],
            active["active_voxel_ids"],
        )

        # C. active history
        active_tracks = self.track_bank.query_active(frame)
        rendered_tracks = self.track_renderer.render(active_tracks, frame)

        # D. Layer-1 graph
        graph1 = self.evidence_graph_builder.build(
            frame, masks, primitives, active_tracks, rendered_tracks
        )
        logits1 = self.edge_scorer_l1(graph1)
        signed_prim_graph = self.signed_graph_assembler.assemble(graph1, logits1)
        prim_cluster_ids = self.layer1_solver.solve(signed_prim_graph)
        current_objects = self.current_object_builder.build(
            prim_cluster_ids, primitives, masks, active_tracks, logits1, graph1
        )

        # E. Layer-2 graph
        assoc_graph = self.assoc_graph_builder.build(current_objects, active_tracks, frame)
        unary_logits = self.unary_scorer(assoc_graph.unary_feat)
        candidate_map = self.candidate_gater.prune(assoc_graph, unary_logits)

        components = self.component_builder.build_components(candidate_map)

        matched = {}
        newborn = []
        for comp in components:
            if comp.is_unambiguous():
                matched.update(self.hungarian_fallback.solve(comp.unary_cost()))
            else:
                pairwise_compat = self.relation_scorer(
                    assoc_graph.obj_edge_feat,
                    assoc_graph.trk_edge_feat,
                    assoc_graph.obj_edge_index,
                    assoc_graph.trk_edge_index,
                    comp.candidate_map
                )
                matched.update(
                    self.graph_matcher.solve(
                        comp,
                        unary_scores=comp.unary_scores,
                        pairwise_compat=pairwise_compat,
                        beam_size=64,
                    )
                )

        # F. update memory
        self.track_bank.apply_assignments(frame.frame_id, matched, current_objects)
        self.track_bank.step_lifecycle(frame.frame_id)

        # G. optional debug package
        return {
            "current_objects": current_objects,
            "matched": matched,
            "tracks_active": list(self.track_bank.active.values()),
        }

J. 阶段推进与停机准则

这部分不是“额外建议”，而是这套实现的一部分。因为你的方法里有几块是真命题，有几块只是合理假设，必须用阶段实验及时止损。

⸻

	•	也不会一上来陷进 fully differentiable graph matching 的工程泥潭
