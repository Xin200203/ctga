下面这版我按“论文方法章节草稿”和“工程实现蓝图”两部分写。先给结论：

我重新核对了 ESAM、AutoSeg3D、OnlineAnySeg、SAI3D、SceneGraphFusion、ODAM、GMTracker、GNN3DMOT、Neural MOT Solver 这些工作后，比较稳的判断是：你的创新点不应该表述成 scene graph representation，也不应该只是给 ESAM 的 affinity 加一个 graph encoder；更合理的表述是，把 online RGB-D 的实例关联分成两个图问题：先用同帧 cross-modal evidence graph 修复当前观测，再用 current-object graph 和 memory-track graph 做二阶图匹配。 ESAM/AutoSeg3D 都还是一阶的 current-to-memory 匹配；SceneGraphFusion/SAI3D/MaskClustering 提供了“同帧或多视图图聚合”的证据；ODAM、GMTracker、Neural Solver 则给了“把 association 本身做成图推理/图匹配”的直接先例。 ￼

⸻

1）正式的方法章节草稿

1.1 方法故事

我们考虑 online RGB-D 室内场景重建中的 class-agnostic object-centric mapping。难点不是只把当前 mask 和历史 track 做一次配对，而是：当前帧本身已经因为 2D/3D 观测不完整而存在 over-seg / under-seg；如果直接做一阶匹配，错误会被写进 memory，后面越来越难纠正。ESAM 的核心是 geometric-aware query lifting、dual-level transformer decoder 和 similarity matrix + bipartite matching；AutoSeg3D 则把问题改写成 instance tracking，引入 LTM/STM/SCL，但 temporal association 仍然是 confidence-gated affinity + Hungarian。它们都很强，但本质仍是一阶 current-to-memory association。 ￼

因此本文的核心思想是两步：
	1.	层 1：same-frame cross-modal evidence graph
利用当前帧 2D masks、当前帧 3D primitives、历史 active tracks 共同修复当前观测，把碎片化 primitive 合成更稳定的当前对象假设。
	2.	层 2：current-object graph ↔ memory-track graph matching
在当前对象和历史 tracks 之间，不只看一阶相似度，还显式利用对象间关系做二阶匹配，从而减少重复物体场景下的 ID switch 和误匹配。

这个故事和 SceneGraphFusion 的“same-part relation + incremental fusion”、SAI3D 的“superpoints + pairwise affinity + region growing”、以及 GMTracker 的“association 应该写成图匹配而不是只做 bipartite matching”是连续的；但我们把它们首次组合到了 online RGB-D object-centric mapping 的设定里。 ￼

⸻

1.2 问题定义

给定在线 RGB-D 输入流
\mathcal X_{1:t}=\{(I_\tau,D_\tau,\mathbf T_\tau)\}_{\tau=1}^{t},
其中 I_\tau 为 RGB 图像，D_\tau 为深度，\mathbf T_\tau 为相机位姿。系统维护一个对象级 memory
\mathcal T_{t-1}=\{\tau_k\}_{k=1}^{K_{t-1}},
其中每个 \tau_k 表示一个长期 3D instance track，包含其几何支持、聚合外观、年龄、可见历史与关键视角缓存。

在时刻 t，前端产生两类当前观测：
	•	2D class-agnostic masks
\mathcal M_t=\{m_i\}_{i=1}^{N_t^M}
	•	当前 active frustum 上的 3D geometric primitives
\mathcal P_t=\{p_j\}_{j=1}^{N_t^P}

我们的目标是同时估计：
	•	当前 primitives 的对象划分
z_t=\{z_j\}_{j=1}^{N_t^P}
	•	当前对象到历史 tracks 的赋值矩阵
X_t \in \{0,1\}^{R_t\times(K_{t-1}+1)}
其中 R_t 是当前局部对象数，额外一列表示 newborn。

我们做如下因子化：
p(z_t, X_t \mid \mathcal M_t,\mathcal P_t,\mathcal T_{t-1})
\approx
p(z_t \mid \mathcal M_t,\mathcal P_t,\mathcal T_{t-1}^{act})
\cdot
p(X_t \mid \mathcal O_t(z_t), \mathcal T_{t-1}^{act}),
其中 \mathcal T_{t-1}^{act} 只保留当前视锥附近和最近活跃的 tracks。这个分解的含义是：先修复当前观测，再做 tracking。它和 SAI3D/SceneGraphFusion 把 primitive 当稳定原子的做法一致，也符合 ODAM 这类 frame-to-model association 系统的分层思路。 ￼

⸻

1.3 层 1：同帧 cross-modal evidence graph

1.3.1 为什么节点必须落在 3D primitives 上

第一层图里，我们不直接优化 2D mask 的实例标签，也不直接让历史 tracks 参与聚类；真正的优化变量只放在 3D primitives 上。原因是：单帧 RGB-D 只看到物体的可见表面，2D masks 视角敏感，历史 tracks 则是长期记忆；最稳定、最适合作为当前优化原子的，是当前 active frustum 里的几何 primitives。SceneGraphFusion 用 geometric segments of primitive shapes 作为 scene nodes，并显式学习 same-part；SAI3D 则先把 3D 点云过分割成 superpoints，再用 2D masks 支持它们的逐步合并。 ￼

1.3.2 图定义

构建异构证据图
G_t^{E}=(V_t^E,E_t^E),\qquad
V_t^E=\mathcal M_t \cup \mathcal P_t \cup \mathcal T_{t-1}^{act}.

边只保留四类：
	•	E_{MP}：mask–primitive，同帧投影支持边
	•	E_{PT}：primitive–track，历史几何支持边
	•	E_{MT}：mask–track，历史重投影支持/否决边
	•	E_{PP}：primitive–primitive，局部几何邻接边

其中 E_{MP} 必须 many-to-many，不能 1:1。因为多个 primitives 可能本就来自同一物体，而一个大 2D mask 也可能错误覆盖多个物体。MaskClustering 的核心 insight 就是：same-instance 不是简单 pairwise IoU，而要结合多视角 consensus；SceneGraphFusion 的 same-part 和 SAI3D 的 superpoint affinity 也都在利用这种“二阶共同支持”。 ￼

1.3.3 边分数

(a) mask–primitive 边

a_{ij}^{MP}
=
\alpha_1\,\text{Cov}\!\left(\pi(p_j),m_i\right)
+\alpha_2\,\text{Contain}_{m\rightarrow p}
+\alpha_3\,e^{-d^{depth}_{ij}/\sigma_d}
+\alpha_4\,s^{app}_{ij}
-\alpha_5\,b_{ij}.

这里我建议特意引入 非对称包含 项，而不是只用 IoU。理由来自 OnlineAnySeg：它用 voxel hashing 快速找候选，再基于可见性修正后的 directed containment ratio 来建 contain matrix，并显式剔除 boundary voxels。对于 partial observation，这个分数比对称 IoU 更合理。 ￼

(b) primitive–track 边

a_{jk}^{PT}
=
\beta_1\,\text{Vote}(V_j,\tau_k)
+\beta_2\,\text{IoU}_{3D}(B_j,B_k)
+\beta_3\,s^{geo}_{jk}
-\beta_4\,d^{center}_{jk}.

其中 \text{Vote} 表示 primitive 的体素支持对历史 track 的空间投票；这和 OVI-MAP 的 spatial voting / super-point registration 思路一致，只不过这里落在 primitive-to-track 上。OVI-MAP 的关键结论之一正是：instance mapping 可以先类无关地做好，再把语义后置。 ￼

(c) mask–track 边

a_{ik}^{MT}
=
\gamma_1\,\text{IoU}_{2D}(m_i,\hat m_k^t)
+\gamma_2\,s^{app}_{ik}
+\gamma_3\,v_{ik}^{vis}.

这里 \hat m_k^t 是将历史 track \tau_k 重投影到当前帧的可见区域。它不是直接决定最终归属，而是提供“支持或否决”信号：如果一个 mask 同时强覆盖两个历史上长期分离的 tracks，它更可能是 under-seg，而不是一个可信的合并依据。

(d) primitive–primitive 边

a_{jj'}^{PP}
=
\delta_1\,\text{Adj}(p_j,p_{j'})
+\delta_2\,s^{normal}_{jj'}
+\delta_3\,s^{color}_{jj'}
-\delta_4\,\text{BoundaryPenalty}.

这条边只是弱先验，不能独裁；否则相邻物体很容易被几何连续性误并。这个结论与 hybrid 2D/3D 方法的共同经验一致。 ￼

1.3.4 证据消元到 primitive graph

第一层不直接聚类异构节点，而是把 masks 和 tracks 当作因子节点，消元到 primitive–primitive signed graph 上：

W_{jj'}^{+}
=
\lambda_{pp} a_{jj'}^{PP}
+
\lambda_m \sum_i \tilde a_{ij}^{MP}\tilde a_{ij'}^{MP}
+
\lambda_t \sum_k \tilde a_{jk}^{PT}\tilde a_{j'k}^{PT},

W_{jj'}^{-}
=
\lambda_c \sum_{k\neq l}
\tilde a_{jk}^{PT}\tilde a_{j'l}^{PT}
\mathbf 1[\tau_k,\tau_l\text{ 空间上明显分离}]
+
\lambda_u\,\text{MaskConflict}_{jj'}.

于是得到 signed primitive graph
G_t^{P}=(\mathcal P_t, W^+, W^-).

这一步的直觉是：
	•	共享 2D 支持，倾向于合并；
	•	共享历史 track 支持，倾向于合并；
	•	分别强支持到两个长期分离的历史 tracks，倾向于拆开。

这是 SceneGraphFusion 的 same-part、SAI3D 的 pairwise affinity、MaskClustering 的 view consensus 在 online 场景里的统一写法。 ￼

1.3.5 第一层目标函数

令 y_{jj'}=1 表示 primitive p_j,p_{j'} 属于同一当前对象，则优化可写为 signed graph partition / correlation clustering：

\min_{y}
\sum_{j<j'} W_{jj'}^{+}\,\mathbf 1[y_{jj'}=0]
+
\sum_{j<j'} W_{jj'}^{-}\,\mathbf 1[y_{jj'}=1].

这本质上就是 multicut / correlation clustering。GASP 明确指出：signed graph partition 的 canonical formulation 就是 multicut / correlation clustering，它是 NP-hard，但已有很多近似求解器；signed 图的好处是 attraction 和 repulsion 同时存在，不需要你事先指定簇数。 ￼

1.3.6 第一层训练损失

第一层我建议只学习边打分，不把离散 partition 也做成端到端可微。v1 的损失写成：

\mathcal L_{P}
=
\mathcal L_{pair}^{PP}
+\lambda_{mp}\mathcal L_{edge}^{MP}
+\lambda_{pt}\mathcal L_{edge}^{PT}
+\lambda_{mt}\mathcal L_{edge}^{MT}.

其中：
	•	\mathcal L_{pair}^{PP}：primitive-pair same-instance 二分类损失
	•	\mathcal L_{edge}^{MP},\mathcal L_{edge}^{PT},\mathcal L_{edge}^{MT}：辅助边监督

这些标签都可以由 GT instance 与当前 geometric over-segmentation 自动生成，SceneGraphFusion 的 same-part label generation 就是成熟先例。 ￼

⸻

1.4 层 2：current-object graph ↔ memory-track graph matching

1.4.1 为什么第二层才是真正的 graph matching

第一层结束后，得到当前对象集合
\mathcal O_t=\{o_r\}_{r=1}^{R_t}.

这时再做 tracking。与 ESAM/AutoSeg3D 不同，我们不再把“当前单个 mask/query”直接和历史 memory 做一阶配对，而是为当前对象和历史 tracks 各建一个图：

G_t^{O}=(\mathcal O_t,E_t^{O}),\qquad
G_{t-1}^{T}=(\mathcal T_{t-1}^{act},E_{t-1}^{T}).

这时节点已经从“view-specific 观测碎片”变成“局部对象假设”，规模和噪声都降下来了。ODAM 已经证明了 posed 视频里的 frame-to-model association 可以用 GNN；GMTracker 则更进一步，明确指出传统 bipartite matching 只利用一阶 affinity，而一般图匹配才能显式使用同侧实体之间的关系。 ￼

1.4.2 一阶 unary score

对于当前对象 o_r 和历史 track \tau_k，定义
\theta_{rk}
=
\eta_g\,\text{Geom}(o_r,\tau_k)
+\eta_a\,\text{App}(o_r,\tau_k)
+\eta_h\,\text{Hist}(o_r,\tau_k).

其中：
	•	\text{Geom}：体素投票、3D bbox 相容性、中心距离
	•	\text{App}：对象聚合特征与 track 聚合特征的相似度
	•	\text{Hist}：来自层 1 的 primitive-track / mask-track 累积支持

1.4.3 二阶 relation score

对当前对象对 (o_r,o_s) 和历史 track 对 (\tau_k,\tau_l)，定义关系边特征：
e_{rs}^{O},\quad e_{kl}^{T}
包含相对位移、相对高度、支撑/接触、互斥关系、投影视角中的深度顺序等。

定义关系兼容分数
\psi_{rs,kl}
=
\text{sim}(e_{rs}^{O},e_{kl}^{T})
-
\text{viol}(e_{rs}^{O},e_{kl}^{T}).

这个项正是 graph matching 相对 Hungarian 的本质增益：当两个椅子的 unary 相似度很接近时，它们与桌子、墙面、显示器的相对关系可以打破歧义。GMTracker 的核心观点就是：Sinkhorn/Hungarian 只是一种 learnable bipartite matching，并没有显式利用 intra-frame relationship；因此它把 data association 写成 graph matching，并用连续 relaxation 的 QP 层和 GST 做加速。 ￼

1.4.4 第二层目标函数

令 X_{rk}\in\{0,1\} 表示当前对象 o_r 赋给历史 track \tau_k，则目标函数为：

\max_X
\sum_{r,k}\theta_{rk}X_{rk}
+
\lambda
\sum_{r<s}\sum_{k<l}
\psi_{rs,kl} X_{rk}X_{sl}

约束为：
\sum_k X_{rk}=1,\qquad
\sum_r X_{rk}\le 1.

加一列虚拟 newborn 即可支持新生对象。

这是标准的二阶 graph matching / QAP 形式。只有当你真的用到了这一项 \psi，论文里才应该写 graph matching；如果最后只做 graph-encoded Hungarian，更准确的说法是 graph-structured association。GMTracker 对这一点的区分很清楚。 ￼

1.4.5 第二层训练损失

v1 不建议把 QAP 层端到端可微训练到底。更稳的是只学习 unary 和 pairwise scorer：

\mathcal L_{A}
=
\mathcal L_{unary}
+\lambda_{\psi}\mathcal L_{pairwise}
+\lambda_{cons}\mathcal L_{consistency}.
	•	\mathcal L_{unary}：对候选 (o_r,\tau_k) 做 match / non-match 二分类
	•	\mathcal L_{pairwise}：对 (o_r,o_s,\tau_k,\tau_l) 做 relation-compatibility 二分类或回归
	•	\mathcal L_{consistency}：约束 layer-1 输出对象与 layer-2 赋值的一致性

如果以后要追求 fully differentiable，可以参考 GMTracker 的 convex QP relaxation + KKT / implicit differentiation，或者 Neural Solver 的“直接预测 graph final partition”；但对你的 indoor RGB-D mapping v1，这条路太重，也缺少同设定验证。 ￼

1.5 推理算法：哪些稳，哪些可试，哪些应避免

1.5.1 已验证、我建议 v1 采用的

层 1：稀疏 signed graph partition。
推荐用 GASP 风格的 agglomerative signed graph partition，或者等价的 graph contraction / approximate multicut。它们在 instance segmentation 里是被反复验证过的，优点是：支持 attraction + repulsion、可稀疏图、可近似在线。 ￼

层 2：候选剪枝后的小规模二阶图匹配。
先做 active-track gating，再把候选 bipartite 图分解成 connected components；小组件上跑二阶 graph matching/QAP，大组件退化成 Hungarian。GMTracker 的 GST 恰恰说明：graph matching 本身是有价值的，但一定要靠 gating 和组件分解加速，否则会慢得不可接受。 ￼

1.5.2 理论上可行，但我不建议作为 v1 主路

可微 QP graph matching layer。
GMTracker 已经证明这是可行的；但它主要验证在 MOT/image matching/point registration 上，而不是 online indoor RGB-D instance mapping。它更适合作为 v2 或 ablation 分支。 ￼

Neural solver / dynamic graph MPNN。
Neural MOT Solver、TrackMPNN 都说明：可以让网络直接在图上预测最终 partition，甚至在多时刻图上修正历史错误。理论上很诱人，但工程复杂度高、训练稳定性差、对数据和轨迹标签要求更重。 ￼

1.5.3 我明确建议避免的

避免 1：每帧对全局所有 tracks 做完整 QAP。
GMTracker 明确指出，图匹配在对象数增大时会变得低效，因此才设计 GST 限制 feasible region。你的 indoor memory 会随时间增长，不做 active-set 剪枝会很快失控。 ￼

避免 2：在第一层直接对所有异构节点做完整 multicut。
第一层真正稳定的原子应是 primitives，masks 和 tracks 更适合作为因子节点；否则图太噪、太大，也不符合 SceneGraphFusion / SAI3D 的经验。 ￼

避免 3：把 open-vocabulary semantics 提前做成主关联信号。
OVI-MAP 的经验更支持“instance mapping 先类无关做好，再后置语义推理”。在当前阶段，语义应是弱辅助或后置层，不应主导 tracking。 ￼

避免 4：直接沿用 OnlineAnySeg 的阈值式 merge 作为最终核心算法。
OnlineAnySeg 的空间哈希和 visibility-aware containment 非常值得借，但作者自己在 repo 里也把“replace the threshold-based mask merging strategy with an adaptive approach”列为 TODO。它更适合做候选生成和几何证据层，而不是你方法的最终决策层。 ￼

1.6 与 ESAM / AutoSeg3D 的对比表述

这一段在论文里只该占一个小节。

ESAM：贡献在于 geometric-aware query lifting、dual-level transformer decoder、以及高效 similarity matrix + bipartite matching；它解决的是“如何把 2D masks 快速提升为高质量 3D queries 并做一阶 online merging”。 ￼

AutoSeg3D：把问题重写成 tracking，引入 LTM、STM 和 SCL；但其 long-term association 仍是 confidence-gated affinity + Hungarian，SCL 也主要是 learning-based mask integration，不是显式的二阶 graph matching。 ￼

我们的区别：
不是更强的 query decoder，也不是更大的 memory；而是把 online association 显式拆成 “观测修复图” + “时序图匹配” 两层，其中第二层是真正的二阶 graph matching。

2）工程实现蓝图

2.1 应该从头写，还是在现有代码上改

我的建议是：

不要从零写系统；也不要直接在 AutoSeg3D 上硬改。
最稳的做法是：
	•	以 ESAM 作为运行时/数据/评测 scaffold；
	•	把 OnlineAnySeg 当作空间索引和几何证据模块的参考实现；
	•	把你自己的核心方法写成一个独立仓库/独立模块层。

理由很具体：
	1.	ESAM 已经支持 custom dataset，并提供 StreamDemo.run_single_frame(rgb, depth, pose, intrinsic) 这一类很实用的在线接口；同时它在 ScanNet200、SceneNN、3RScan 上都有现成结果与工程脚手架。 ￼
	2.	AutoSeg3D 虽然很强，但官方 README 也明确说它建立在 ESAM codebase 上，并继承了随机采样和 BatchNorm 训练不稳定等 legacy issues。它适合作为强 baseline，不适合作为你第一版方法的主底座。 ￼
	3.	OnlineAnySeg 的 voxel hashing、boundary voxel 处理、visibility-aware contain matrix 非常值得借；但它当前主 merge 逻辑仍是阈值式策略，作者也把“改成 adaptive approach”列为 TODO。 ￼

所以工程策略应该是：新建你的主仓库，ESAM 和 OnlineAnySeg 作为 third_party/reference，而不是直接魔改其中之一。

⸻

2.2 主体方法的模块划分

我建议主系统拆成 6 个核心模块，外加 1 个可选语义头。

模块 A：在线输入与活动视锥地图

输入：rgb, depth, pose, intrinsic
输出：当前 active frustum 点/体素、全局稀疏地图状态

这层直接复用 ESAM 的在线数据流接口最省时间。 ￼

模块 B：2D front-end

输入：当前 RGB
输出：class-agnostic masks + 2D embeddings

v1 建议先离线预计算，避免前端波动影响关联实验。ESAM/OnlineAnySeg 都依赖 2D foundation model 输出，但你的论文创新不在这里。 ￼

模块 C：3D primitive builder

输入：当前 active frustum 点云/体素
输出：当前 3D primitives

v1 先用纯几何 over-segmentation，不要急着上 learned 3D segmenter。因为你的核心是 association，不是 proposal recall。这个选择与 SAI3D、SceneGraphFusion 的 primitive-first 经验一致。 ￼

模块 D：层 1 evidence graph builder + scorer

输入：masks、primitives、active tracks
输出：signed primitive graph

这里要显式实现：
	•	mask–primitive 边
	•	primitive–track 边
	•	mask–track 边
	•	primitive–primitive 邻接边

其中 OnlineAnySeg 的 voxel hash / visibility-aware contain ratio 可以直接借做候选生成和 directed containment feature。 ￼

模块 E：层 1 primitive partition solver

输入：signed primitive graph
输出：当前局部对象 \mathcal O_t

v1 用 GASP 风格 agglomerative signed partition 即可，不必自己造轮子。GASP 官方 repo 直接提供 run_GASP(graph, signed_edge_weights, ...)。 ￼

模块 F：层 2 graph matcher + track bank

输入：当前对象 \mathcal O_t、active track set
输出：匹配、新生、休眠、更新后的 track bank

这里实现：
	•	unary scorer
	•	relation scorer
	•	active-set candidate pruning
	•	小组件 graph matching，大组件 Hungarian fallback
	•	birth / death / dormant / reactivate 规则

可选模块 G：后置语义头

输入：稳定 tracks 的关键视角
输出：CLIP / VLM 语义标签

这层完全可以后置。OVI-MAP 的 decoupled 设计很支持这种分离。 ￼

	•	frontends/：2D masks 与 embedding 读取/缓存
	•	mapping/：active frustum、体素索引、track reprojection
	•	primitives/：几何 over-segmentation 与 primitive feature
	•	graph1_evidence/：层 1 边构建、edge scorer、signed graph 组装

	•	memory/：track bank、生命周期管理

2.4 推进节奏：必须分阶段，不要一口气做完

阶段 0：跑通两个参考仓库

先复现 ESAM 和 OnlineAnySeg。

目的不是追数，而是确认你已经掌握：
	•	ESAM 的 online loop、custom data 接口、评测入口。 ￼
	•	OnlineAnySeg 的 main.py -> insert_seg_frame -> MaskGraph.query_mask_under_visible_part -> update_masks 这条空间关联链。其 repo 明确包含 MaskGraph.py / Scene_rep.py / voxel_hashing.py / main.py。 ￼

阶段 1：只做层 1，不做 tracking graph

先验证：
同帧 evidence graph 是否真的能修复 over-seg / under-seg。

这一步只评估：
	•	当前局部对象 mask quality
	•	fragmentation
	•	merge error

如果这一层都没有明显收益，就不要继续做层 2。

阶段 2：加 track bank，但只做 unary Hungarian

这一步的目的是建立一个你自己的强一阶基线。
也就是：同样的 layer-1 输出对象，但 temporal stage 只用 unary + Hungarian。

这样你后面所有二阶 graph matching 的收益，才是干净可解释的。

阶段 3：在 active-set 上加二阶图匹配

只在小 connected components 上打开二阶项。
先看两个指标：
	•	ID switches
	•	重复类物体场景下的 association accuracy

如果没有显著收益，再考虑 relation feature 是否设计错了，而不是立刻扩大模型。

阶段 4：学习式边打分

在手工分数跑通之后，再把层 1 和层 2 的 scorer 变成小型 MLP / GNN。

阶段 5：后置语义

最后再接 open-vocab 语义。

2.5 哪些是已知，哪些是假设，哪些是必须尽快验证的

已知

这些在已有工作里已经相当明确：
	•	primitive / superpoint 是修复 over-seg 的合适原子。 ￼
	•	same-part / view-consensus 这类高阶支持，对 merge 很重要。 ￼
	•	一阶 Hungarian 是强基线，但会忽略 intra-view relation。 ￼
	•	graph matching/QAP 能利用对象间关系，但必须做 gating 才能跑得动。 ￼

假设

你的方法依赖这些前提：
	•	场景基本静态；
	•	pose 质量足够好；
	•	当前 active set 不会太大；
	•	同类重复物体足够多，关系项才有价值；
	•	当前碎片的主要问题是 over-seg，而不是 proposal recall 崩掉。

必须尽快验证的

我建议尽早做 3 组阶段实验：
	1.	layer-1 only 是否显著降低 fragmentation
	2.	same objects + unary Hungarian 和 same objects + graph matching 的差距
	3.	relation feature 到底该用哪些：仅相对位置，还是要加支撑/遮挡/接触

如果这三组里有一组不成立，就要及时改模型，而不是继续堆模块。

2.6 baseline 与实验设置

主 baseline
	•	ESAM / ESAM-E：最直接的一阶 online baseline。 ￼
	•	AutoSeg3D：当前更强的一阶 tracking-centric baseline。 ￼

参考 baseline
	•	OnlineAnySeg：空间关联与 hash-based overlap 的参考，但任务侧重点不同。 ￼
	•	SAI3D / SceneGraphFusion：同帧 primitive graph 设计参考。 ￼
	•	ODAM / GMTracker：temporal graph association 设计参考。 ￼

数据集

优先：
	•	ScanNet200
	•	ScanNet
	•	SceneNN
	•	3RScan

这与 ESAM / AutoSeg3D 的主评测线保持一致。 ￼

指标

除了 AP / AP50 / AP25，还必须加：
	•	fragmentation
	•	merge error
	•	ID switches
	•	latency / FPS

因为你的主张不是“点级分割更细一点”，而是“association 更正确”。

最后一句判断

如果目标真的是 CVPR 级别，我认为最值得押注的版本不是“更强 decoder”，也不是“最终长成 concept graph”，而是这条线：

Layer-1 同帧 evidence graph 修复观测 + Layer-2 current-object / memory-track 二阶图匹配。

这条线的 novelty、数学形式、工程可行性和实验可拆解性，目前是最平衡的。

下一步最合适的是我直接把这套方法继续往下落成一版伪代码级别的实现文档：类名、函数签名、张量形状、训练数据生成方式、以及每个模块的 ablation 开关。
