# 核心假设与风险

## 关键假设

- 场景基本静态，主体变化不剧烈。
- 相机位姿质量足够稳定，重投影与几何支持可信。
- 当前 active set 可控，不会让 graph matching 复杂度失控。
- 目标场景存在足够多的重复物体，关系项才有区分价值。
- 当前误差瓶颈主要来自 over-seg / under-seg 与关联错误，而不是 proposal recall 崩坏。

## 主要风险

### Layer-1 风险

- primitive 质量不足，导致图修复空间本身就不稳定。
- `MT` 与 `PT` 证据噪声过大，把历史错误反向注入当前帧。
- signed graph 权重设计不合理，造成过度合并或过度切分。

### Layer-2 风险

- relation feature 不足以提供超越 unary 的判别力。
- active track set 过大时，QAP 或 beam search 仍然开销过高。
- candidate gating 过严会截断真阳性，过松会拖垮求解效率。

### 工程风险

- 如果 2D front-end 波动太大，会掩盖图结构本身的收益。
- 如果 teacher replay 与在线 memory 分布差距太大，训练效果会失真。

## 需要尽早验证的命题

1. Layer-1 是否能稳定降低 fragmentation，同时不明显抬高 merge error。
2. 在同一批 current objects 上，graph matching 是否优于 unary Hungarian。
3. relation feature 是否只靠相对位置就足够，还是必须加入支撑、接触、遮挡和深度顺序。
