# 设计决策与取舍

## 为什么 primitive 是第一层优化原子

- 2D masks 受视角影响强，天然不稳定。
- history tracks 属于长期记忆，不适合直接参与当前帧聚类变量。
- 当前 active frustum 中的几何 primitive 是三者里最稳定、最适合做局部图优化的载体。

## 为什么不直接在异构大图上做最终 partition

- 图规模会迅速膨胀。
- mask 与 track 节点噪声比 primitive 更高。
- 工程上更难控制错误来源，不利于调试与分阶段验证。

## 为什么 v1 不做 fully differentiable QAP

- 复杂度和训练稳定性都偏高。
- 当前设定下更重要的是验证问题分解本身，而不是追求端到端形式优雅。
- 先把 scorer 学稳，再谈 differentiable solver 更现实。

## 为什么语义后置

- 项目的主要创新点在 class-agnostic association。
- 提前引入 open-vocabulary semantics 会混淆误差归因。
- 语义更适合作为稳定 track 的后置解释层，而不是主关联信号。
