# 方法总览

## 一句话

CTGA 将 online RGB-D 的实例关联拆成两个连续图问题：先修复当前帧观测，再做当前对象到历史轨迹的图匹配。

## 核心问题

直接把当前 mask 或 query 与历史 memory 做一阶匹配，容易把单帧 over-seg / under-seg 错误写入 memory，并在后续帧持续放大。

## 两层图方法

### Layer 1: Same-frame Cross-modal Evidence Graph

输入：

- 当前帧 2D class-agnostic masks
- 当前 active frustum 中的 3D primitives
- 当前可见的 active tracks

建图方式：

- 节点集合是 `masks + primitives + active tracks`
- 真正的优化原子落在 `3D primitives`
- `masks` 与 `tracks` 主要作为证据因子，不直接参与最终聚类变量

边类型：

- `MP`: mask-primitive，同帧投影与可见性支持
- `PT`: primitive-track，历史几何与体素支持
- `MT`: mask-track，重投影支持或 veto
- `PP`: primitive-primitive，局部邻接与弱几何先验

输出：

- 消元得到 signed primitive graph
- 通过 GASP / multicut / correlation clustering 得到当前局部对象

目标：

- 修复当前帧的 over-seg / under-seg

### Layer 2: Current-object Graph to Memory-track Graph Matching

输入：

- Layer-1 输出的 `CurrentObjectHypothesis[]`
- 当前 active track bank

匹配方式：

- 先计算 unary score
- 再利用对象间关系构建 pairwise compatibility
- 对每个对象只保留 top-k candidate tracks
- 在小连通分量上做 beam-QAP / graph matching
- 大组件退化为 Hungarian fallback

目标：

- 提升 temporal association 稳定性
- 降低重复物体场景下的 ID switch

## 方法边界

当前版本默认：

- 先做 class-agnostic association
- 语义只后置，不主导主关联环节
- 不追求端到端可微 partition 或 QAP
- 不与 SLAM 后端做联合训练

## 对比定位

- 相比 ESAM / AutoSeg3D：区别不在更强 decoder，而在把关联显式拆成“观测修复图 + 时序图匹配”。
- 相比 OnlineAnySeg：借鉴空间候选和 visibility-aware containment，但不采用阈值式 merge 作为最终决策。
- 相比 GMTracker / ODAM：吸收 graph matching 与图式 association 的思想，但落到 online RGB-D object-centric mapping 场景。
